BeyondRealityFaceManager starts the SDK automatically. BeyondRealityFaceManager.INIT will be dispatched right before BRF initializes components, that depend on their vars/params. BeyondRealityFaceManager.READY will be dispatched when BRF started up. Please don't call init directly. That is being done automatically. (You must not create more than one instance of this class. Please reuse this instance!)

BeyondRealityFaceManager has all the result data for the different tracking modes:

BRFMode.POINT_TRACKING: Tracks single or multiple points in a video/webcam stream.

BRFMode.FACE_DETECTION: Finds faces in images/webcam streams and marks them with rectangles.

BRFMode.FACE_TRACKING: Detects faces and analyses them to find detailed face features.

Please see the SDK examples. Implementations for all tracking modes are included.

Author: Marcel Klammer, Tastenkunst GmbH, 2014

BeyondRealityFaceManager()Constructor
public function BeyondRealityFaceManager(src:BitmapData, roi:BRFRegionOfInterest, stage:Stage, params:Object = null)

Initializes the SDK and sets up BRF automatically. Once BRF is ready a BeyondRealityFaceManager.READY event will be dispatched. Just listen to that event.

For questions about the implementation, please refer to the examples, that are included in the SDK.

Parameters
src:BitmapData — mandatory, the BitmapData BRF is working on (default size should be 640x480)
 
roi:BRFRegionOfInterest — mandatory, define the region of interest in your src BitmapData, that should be analysed (should definitly NOT exceed 800x600, default should be 640x480)
 
stage:Stage — mandatory, put your stage referance in here
 
params:Object (default = null) — optinal

reset()method 
public final function reset():void

Resets the tracking process of BRF. Usually to pause BRF tracking or restart with FACE_DETECTION. Calls vars.pointTrackingVars.reset() to remove all tracked points, if vars.mode == BRFMode.POINT_TRACKING.

Don't call this method before READY was dispatched.

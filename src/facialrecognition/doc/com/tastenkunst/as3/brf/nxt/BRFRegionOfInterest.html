InheritanceBRFRegionOfInterest Object

Usually BRF operates on a 640x480 BitmapData with BRFRegionOfInterest being 640x480 as well. But there are cases, where you want to work on a bigger BitmapData, but restrict the BRF analysis to a certain region of interest.

One example: You want to show a 1280x720 camera image, but only want to analyse the center 800x600 area of that BitmapData. Then you can set _brfRoi = new BRFRegionOfInterest(0, 0, 800, 600); and _brfRoi.center();

The region of interest should definitly not exceed 800x600, because BRF would have to process much more pixels than it can in real time. For most purposes 640x480 for BitmapData and ROI should be just fine.

Please see the SDK examples. Implementations for all tracking modes are included.
